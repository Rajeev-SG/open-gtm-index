# Open GTM Index Editorial Evidence

## Purpose and audience

Open GTM Index helps software buyers, operators, and technical teams compare open-source replacements for commercial sales and marketing software. People use it to scan category leaders, inspect ranking evidence, understand licences, and identify which commercial products each project can replace.

## Locked visual direction

The interface is an editorial research publication, not a conventional software dashboard. It should feel independent, exact, calm, and inspectable. Preserve the warm paper-like canvas, dark ink, large serif headline, cobalt actions, green licence status, and amber numeric scores from the approved desktop reference.

Do not add gradients, decorative photography, floating shadows, glass effects, metric hero cards, or oversized rounded containers. Use fine rules, precise corners, disciplined alignment, and dense real data.

## Desktop source composition

- A 68px header with a compact black square mark, product name, four navigation links, and GitHub action.
- A two-line editorial hero with the exact headline “Ranked tools you can inspect, self-host, and improve.”
- Research metadata: “53 projects reviewed · 50 ranked · 13 categories · checked 20 July 2026”.
- Primary button “Explore the rankings”; text action “Read the scoring method”.
- Public-data note stating that restricted or unclear licences are visible but not ranked.
- A four-part evidence strip: 100% open data, Evidence-backed, Verified open-source, Community governed.
- Six visible category-leader cards using real projects and scores.
- A compact ranking table beside a “How the score works” panel.

## Responsive rules

### Tablet

- Keep the product mark and name in the header. Replace the wide navigation with a compact menu button when links no longer fit.
- Preserve the hero hierarchy and buttons. Allow the headline to use three lines if needed.
- Change the evidence strip to a two-by-two grid.
- Show category leaders in a three-column, two-row grid.
- Place the scoring explanation below the full-width ranking table.
- The table may scroll horizontally, but the first project column and score must remain easy to find.

### Mobile

- Use a 16px page gutter and a compact header with menu button.
- Let the headline wrap naturally across three to four lines; do not reduce it to a generic small heading.
- Make the primary button full width and keep the scoring link directly below it.
- Stack the evidence principles as bordered rows.
- Present category leaders in a horizontal snap carousel so users can compare without a very long six-card stack.
- Transform each ranking row into a labelled evidence card. Do not hide category, score, stars, licence, or replacement information.
- Change the score explanation to a three-column grid.
